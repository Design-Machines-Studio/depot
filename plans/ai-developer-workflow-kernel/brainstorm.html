Brainstorm: AI Developer Workflow Kernel

Problem & Intent

Pipeline and dm-review already model sophisticated AI developer workflows, but many deterministic transitions, retry rules, evidence requirements, and cleanup paths remain encoded as prose interpreted by an agent. The approved design adds a dependency-free Python control plane while preserving Markdown policy, current commands, host adapters, and human gates.

Docker TTLAutomatic stale cleanupRemove only Depot-labeled resources older than 24 hoursPrevents clutter without broad or unsafe Docker pruning.
UX verificationDeclared personas are blockingPer-persona scenarios, browser, viewport, screenshot or traceA UI run is incomplete if only the default user was exercised.
Browser recoveryRestart, alternate engine, then human escalationNo silent skip after a required browser node failsTooling snags must not erase visual or UX findings.
ScopeRepo-local engine onlyNo daemon or external ticket intakeKeeps the first implementation bounded and testable.
