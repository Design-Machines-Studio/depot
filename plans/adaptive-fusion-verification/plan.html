Verification Strategy

  1. Run focused unit suites after each chunk and repository-wide workflow-kernel tests once per completed level.
  2. Run tools/validate-workflow-kernel.py, tools/validate-workflow-contracts.sh, routing economics, dependency, generated alias/manifest, dual-compatibility, and full composition checks.
  3. Test legacy manifests and receipts explicitly; absence of new fields must preserve current standard behavior with default provenance.
  4. Test security invariants: decision leverage cannot authorize an untrusted provider, weaken sensitive routing, change workflow class, or mutate cleanup authority.
  5. Test economics invariants: scoped attempts do not duplicate run totals and missing provider data remains missing.
  6. Keep the known external Assembly UX declaration mismatch separate from Depot regressions unless the parser contract changes.
