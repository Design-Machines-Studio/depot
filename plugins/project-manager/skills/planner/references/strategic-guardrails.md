# Strategic Guardrails — Systems Dynamics Check

Load this reference during Phase 8 (Sprint Loading) and Phase 9 (Sprint Commitment).

## Before Committing Sprint Work

Every sprint commitment should pass three checks:

### 1. Loop Alignment

Which feedback loop does this work serve?

| Loop | Serves if... | Current priority |
|------|-------------|-----------------|
| R1: Product-Adoption | Directly ships or improves the active pilot prototype | **PRIMARY — 80% of sprint capacity** |
| R2: Solidarity Ecosystem | Strengthens partner relationships, responds to pilot pipeline | 10% — lightweight updates, warm conversations |
| R3: Propaganda | Creates shareable content from the build process itself | 10% — double-duty artifacts only, no dedicated marketing |
| R4: Knowledge | Research that directly informs a current product decision | As needed — pull from RAG, don't schedule research sprints |

If a task doesn't serve R1-R4, question whether it belongs in this sprint.

### 2. Archetype Traps

Flag any task that matches these patterns:

**Eroding Goals:** "This freelance project is just for a month." → Is this decision serving the mission or reducing financial anxiety? If the latter, note it explicitly and timebox ruthlessly.

**Growth and Underinvestment:** "Let's build the baseplate abstraction before the second pilot." → Has a real pilot validated this need? If not, defer. Ship for the pilot you have, extract patterns after.

**Shifting the Burden:** "I'll take this contract work to cover the burn." → Every hour on non-Assembly work is an hour not shipping the prototype that generates its own revenue. Only accept if it genuinely doesn't compete for Assembly time.

### 3. First-Turn Focus

Until R1 completes its first turn (pilot ships → pilot pays → model proven), apply this filter:

- **Does this task move TACO (or the active pilot) closer to shipping?** → Yes: commit it.
- **Does this task keep a warm pipeline conversation alive?** → Yes: commit it, but keep it small.
- **Does this task produce a shareable artifact from work already being done?** → Yes: commit it as a secondary output.
- **Does this task feel important but doesn't pass the above?** → Backlog it. It's not wrong, it's just not now.
